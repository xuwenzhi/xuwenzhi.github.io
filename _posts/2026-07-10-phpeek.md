**phpeek** is a terminal tool that steps a PHP script one line at a time and shows the
opcodes, the source, the variables, the memory, the call stack, and the engine's own
structs — side by side, live. It's driven by `phpdbg`, so there's no gdb, no C, nothing
to compile.

<!-- more -->

![phpeek stepping a script](/assets/img/phpeek.gif)

```bash
phpeek watch script.php
```

Each `space` executes one more line; every panel is re-queried from the running process
and redrawn. Here's the layout:

```
  phpeek · LIVE   loop.php   line 3   in combine   step 3
  OP_ARRAY  combine   ops 7  args 2  CVs 2  tmps 1  lines 6-8
  OPCODES  > = line running now                 SOURCE
    L0006 0000  CV0($x) = RECV 1                   1  <?php
    L0006 0001  CV1($y) = RECV 2                   2  function add($a, $b) {
  > L0007 0002  T2 = ADD CV0($x) CV1($y)         > 3      return $a + $b;
  > L0007 0003  RETURN T2                          4  }
  VARIABLES
    $x  integer = 2
    $y  integer = 3
  MEMORY  emalloc'd, in use  502,200 B
    entered combine()   +480 B  (call frame on the VM stack)
  CALL STACK  {main} is #0; each call stacks upward
    frame #1: combine(x=2, y=3)
    frame #0: {main}
  [space] step   [r] restart   [s] script   [i] opline+cap   [t] tables   [q] quit
```

## The panels

- **OP_ARRAY + OPCODES + SOURCE** — the compiled opcodes for the current function beside
  the script it came from, `>` marking the line about to run. The `OP_ARRAY` band names
  the compiled unit and its counts (ops, args, CVs, tmps, line span); step into a call
  and it swaps to the callee's.
- **VARIABLES** — the compiled variables (CV slots) and their live values.
- **MEMORY** — bytes in use (`memory_get_usage()`) and the change since the last step,
  attributed to *what caused it*: a value assigned, or a call entered/returned as its
  frame is pushed/popped.
- **CALL STACK** — frames pushing and popping as you step into and out of calls, with
  live argument values. `{main}` is `#0` at the bottom.

## The struct panels

Two toggles overlay the actual engine structs.

Press `i` for the **opline** view. A source line usually compiles to several oplines, so
it opens on the first and the arrow keys walk through them, decoding each into its
`zend_op` fields — and highlighting that exact opline (yellow bar) up in the OPCODES
panel:

```
  ZEND_OP  line 7 -> 2 oplines; showing op 1/2  ([<-/->] to walk)
    opcode          ADD
    op1             [CV ] CV0($x)
    op2             [CV ] CV1($y)
    result          [TMP] T2
    lineno          7   (op #0002 in the op_array)
    handler         ZEND_ADD_SPEC_CV_CV_HANDLER  (SPEC-naming convention)
```

`i` also opens the **zend_execute_data** panel — the current call frame, laid out on its
real offsets:

```
  ZEND_EXECUTE_DATA  current frame — 80-byte header, then CV slots
    +0x00  opline             -> op #0002 T2 = ADD CV0($x) CV1($y)
    +0x18  func               -> combine
    +0x20  This (zval,16B)    -> no $this   [u2 = num_args = 2]
    +0x30  prev_execute_data  -> {main}
    +0x50  CV slots (each a 16B zval, contiguous after the header):
      +0x50  $x = integer = 2
      +0x60  $y = integer = 3
```

As you walk the oplines with the arrows, this panel's `opline` field tracks the cursor,
so you can watch `execute_data->opline` advance one instruction at a time. There's also
`[t]` for the engine tables (user functions and classes, from reflection).

## How it works

phpeek is a single bash script. It launches `phpdbg -qe <script>` as a coprocess and
talks to it over a pair of file descriptors, one command at a time. Everything you see
is reconstructed from text phpdbg already prints — opcodes from `print exec`, variables
from guarded `ev` expressions, the frame from `back`. Nothing reads memory directly, so
it needs no gdb and works against any PHP 8 build. The struct panels are honest
reconstructions: the field *values* are real and the offsets are verified, but they're
labeled as derived rather than pretending to be a raw dump.

## Try it

```bash
git clone https://github.com/xuwenzhi/phpeek
cd phpeek
./phpeek check                       # verify php + phpdbg + bash 4+
./phpeek watch examples/loop.php
```

macOS: `brew install php bash`. Debian/Ubuntu: `apt install php-cli php-phpdbg`.

Source and docs: [github.com/xuwenzhi/phpeek](https://github.com/xuwenzhi/phpeek).
